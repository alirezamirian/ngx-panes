Theming

To create a custom theme, you need to import ngx-panes-theme mixin and include it wherever you want. ngx-panes-theme mixin currently accepts a configuration map (object) with the following keys:

Option Required Description Default
primaryColor check Used for active pane tab -
backgroundColor check Used for pane background color. By default, background color of pane tabs and header will be set to a little darkened version of the backgroundColor. -
tabsContrast close Defines how much tabs background should be darker than background. 3%
headerContrast close Defines how much pane header background should be darker than background. 5%
borderContrast close border color is set to foreground color. borderContrast defines how much the opacity of borders. 20%
lightTextColor close The color to be used for text color, whenever light color is legible. #eee
darkTextColor close The color to be used for text color, whenever dark color is legible. #333
This configuration parameters for theming are experimental and subject to change in future. Wherever ngx-panes-theme is applying a background color to something, it's smart enough to apply a legible light or dark text color based background color.

Style encapsulation

When you define a theme by including ngx-panes-theme mixin, the scope of the theme completely depends on the place you are including the mixin. For example you can define a theme to be applied to all ngx-panes instances, by including the ngx-panes-theme mixin in a global style. You can also include the mixin inside styles of a component and it will be applied to all ngx-panes instances inside that component's view. You can also define several themes within appropriate wrappers.
