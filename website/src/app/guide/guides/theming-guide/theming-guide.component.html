Theming

To create a custom theme, you need to import ngx-panes-theme mixin and include it in whatever scope you want it to get applied: Theming demo shows it in action.

ngx-panes-theme mixin currently accepts a configuration map (object) with the following keys:

Option Required Description Default
{{option.name}} {{option.required ? 'check' : 'close'}} {{option.defaultValue}} -
This configuration parameters for theming are experimental and subject to change in future. Wherever ngx-panes-theme is applying a background color to something, it's smart enough to apply a legible light or dark text color based background color.

Theme scope

When you define a theme by including ngx-panes-theme, the scope of the theme application depends on the place you include the mixin.

Single global theme

You can define a theme to be applied to all ngx-pane-area instances, by including the ngx-panes-theme mixin in a globally applied style.

If you are using Angular CLI, you can put it inside styles.scss (which is created by Angular CLI by default), or any other file which is included in styles property of your app inside .angular-cli.json file. Learn more.

Inside component styles

You can also include ngx-pane-theme mixin inside styles of some component, But note that the generated style is subject to that component's view encapsulation. Unless the view encapsulation is set to None, you need to include the mixin inside a Shadow Piercing combinator (::ng-deep, /deep/ or >>>). However it's deprecated and you don't need to take this approach in most cases.
