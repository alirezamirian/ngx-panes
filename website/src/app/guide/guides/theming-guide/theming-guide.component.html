Theming

To create a custom theme, you need to import ngx-panes-theme mixin and include it in whatever scope you want it to get applied: Theming demo shows it in action.

ngx-panes-theme mixin currently accepts a configuration map (object) with the following keys:

Option Required Description Default
{{option.name}} {{option.required ? 'check' : 'close'}} {{option.defaultValue}} -
This configuration parameters for theming are experimental and subject to change in future. Wherever ngx-panes-theme is applying a background color to something, it's smart enough to apply a legible light or dark text color based background color.

Theme scope

When you define a theme by including ngx-panes-theme mixin, the scope of the theme depends on the place you are including the mixin.

Single global theme

You can define a theme to be applied to all ngx-pane-area instances, by including the ngx-panes-theme mixin in a globally applied style.

Scoped to some component view

You can also include the mixin inside styles of a component and it will be applied to all ngx-pane-area instances inside that component's view (Assuming you are not disabling Angular's view encapsulation ).
